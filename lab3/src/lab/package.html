The package contains the code for studying a number of structural patterns.

Introduction

In this lab, you will make the graphics of a snake game flexible by using structural design patterns. The game will be extended with graphical modules that you will make usable. In the beginning of every section you must write a short text, about 2–5 sentences, explaining your choice of design pattern. Ask your lab assistant if you cannot find a design pattern that fits. The graphical modules are constructed so that it will take some thinking to make them usable in the game.

The Eclipse project for this lab can be set up in the same way as it was described with respect to the first lab. As before, you are supposed to complete the implementation of the project by filling in the gaps marked with

// YOUR CODE HERE

in the source files.

Usage

The snake can either be controlled by the arrow keys on the keyboard or by an automatic demo controller. There are also several possible graphical modules to use for displaying the snake, background, and bonuses.

Purpose

The purpose is to learn how to code with structural patterns and choose from a pattern dictionary the pattern most suited in a given situation.

Readings

The chapters on structural design patterns in GoF on p. 137–217.

Assignments

First, make ControllerKey usable. Edit setKeyControll in Snake.java.

Second, Renderer is a common interface of all Graphics modules. Graphics3D has totally different method names than the Renderer interface. Therefore it needs to be adapted (hint intended).

Third, ModuleA, ModuleB, and ModuleC have the functionality for displaying a background, snake, and bonus, but they have totally different interfaces than the Renderer. Moreover, each module does one thing, but you need one object that can do all three things. Hint: Façade?

Fourth, decorate (subtle hint!) a renderer such that you can display the shadow of the snake and of the bonus. The shadow applies to the current renderer, regardless if it is the original, 3D, or ABC.

Hand In

Apart from explaining the details of your implementation and answering the questions of your lab assistant, you are required to hand in the classes that you have created and those that you have modified. To this end, provide your code with comments where it is needed, compress all the relevant classes (*.java only) into a single archive, and send this archive to the lab assistant.
